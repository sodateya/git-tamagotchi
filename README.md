# 🥚 Git Tamagotchi

GitHub の活動（コミット・PR・レビュー・Issue）でデスクトップ上のペット／植物が育つ、常駐型の育成ガジェットです。エンジニアの成果を「お世話」に変換して、楽しく成果を出すためのツール。

```
コミット    = エサ／水やり   （5pt）
PR 作成     = 成長イベント   （20pt）
レビュー    = 特別ボーナス   （30pt・他者貢献を厚く評価）
Issue 対応  = おやつ         （10pt）
```

累計ポイントでステージが上がり、直近の活動状況で「元気度」が変化します。活動が続くと🔥連続日数が伸び、しばらく放置するとお腹を空かせます。

## キャラクターとステージ

| ステージ | ペット | 植物 | ネコ | スライム | キノコ |
|---|---|---|---|---|---|
| 0pt | 🥚 たまご | 🌰 たね | 🐾 まめ | 💧 しずく | 🟤 ほうし |
| 30pt | 🐣 ひな | 🌱 め | 🐱 こねこ | 🟢 ちびすら | 🍄 めばえ |
| 120pt | 🐥 こども | 🌿 わかば | 😺 ねこ | 🫧 スライム | 🍄 きのこ |
| 350pt | 🐔 おとな | 🌷 かいか | 🐈 りっぱ | 👑 こうてい | 🍄 おおきのこ |
| 800pt | 🦅 達人 | 🌳 大樹 | ✨ しんぴ | 🌟 だいおう | ✨ まほうのこ |

## お世話システム

時間経過でお世話状態が溜まります。対応するアクションで解消されます。

| お世話 | 解消アクション |
|---|---|
| 🍗 空腹 | コミット |
| 💧 水不足 | Issue 対応 |
| 💩 うんち | PR 作成 |
| 🌿 くさ | レビュー |

## 必要なもの

- Node.js 18 以上（推奨: 20 以上）
- GitHub アカウント

## セットアップ

```bash
# 1. 依存をインストール
npm install

# 2. 起動
npm start
```

起動するとデスクトップに小さな筐体が浮かび、タスクトレイ（メニューバー）にアイコンが出ます。

## GitHub トークンの設定

1. トレイアイコン → 「設定を開く…」
2. 設定画面の「トークンを発行する →」リンクから GitHub のトークン作成ページへ
3. 権限は **read:user** のみでOK（fine-grained トークンなら読み取り専用で十分）
4. 発行したトークンを貼り付けて「保存する」

トークンは外部に送信されず、`~/.git-tamagotchi/config.json` にこのPC内だけで保存されます。

## 操作

- **筐体をドラッグ**: 好きな位置へ移動
- **↻ ボタン**: いますぐ最新の活動を取得（自動でも30分ごとに更新・リアルタイムポーリングあり）
- **@ ボタン**: 自分の GitHub プロフィールを開く
- **? ボタン**: お世話ガイド（ヘルプ）を開く
- **トレイメニュー**: キャラの切り替え、更新、設定、終了

### 隠しキーシーケンス

筐体にフォーカスした状態でキーを順番に入力すると特殊画面が開きます。

| 入力 | 効果 |
|---|---|
| `tamago` | 全キャラ・全ステージ確認デバッグ画面 |
| `shinka` | 進化条件シート |

## 設定画面でできること

- **育てるキャラクター**: ペット / 植物 / ネコ / スライム / キノコ
- **シェルのカラー**: ピンク・きいろ・スカイ・みどり・むらさき・オレンジ・しろ・こん
- **シェルの柄**: 無地 / 水玉 / ストライプ / チェック
- **ウィンドウサイズ**: スライダーでリアルタイム調整
- **透過度**: スライダーでリアルタイム調整
- **リセット**: 育成データを初期化してたまごからやり直し

## ファイル構成

```
git-tamagotchi/
├── main.js              Electronメイン（ウィンドウ/トレイ/定期更新）
├── preload.js           レンダラーへ安全にAPIを公開
├── src/
│   ├── characters.js    キャラクターSVG定義
│   ├── github.js        GitHub GraphQL / Events API でアクティビティ取得
│   ├── growth.js        育成ロジック（スコア/ステージ/元気度）← 数値はここで調整
│   ├── index.html       キャラ表示（たまごっち風LCD）
│   ├── renderer.js      キャラ描画の更新・キーシーケンス検知
│   ├── settings.html    設定画面
│   ├── settings.js      設定ロジック
│   ├── help.html        お世話ガイド
│   ├── debug.html       全キャラ確認デバッグ画面
│   ├── debug.js         デバッグロジック
│   ├── shinka.html      進化条件シート
│   └── shinka.js        進化条件表示ロジック
```

## カスタマイズの入り口

- **ポイント配分やステージのしきい値**: `src/growth.js` の `POINTS` と `STAGES`
- **キャラの見た目**: `src/characters.js` の各キャラSVG定義
- **元気度の判定**: `src/growth.js` の `streakAndMood`
- **お世話の減衰速度**: `src/growth.js` の `CARE_DEFS`
- **筐体の色・アニメーション**: `src/index.html` の `<style>`

## 今後の発展アイデア

- PR の「マージ済み」を別カウントして大きな成長イベントに（GraphQL の検索APIを追加）
- リポジトリ別／言語別に違う種類のペットを育てる
- 連続日数のごほうび進化、レアな見た目の解放
- 複数キャラを並べてチームの活動を表示
- 配布したい場合は electron-builder でインストーラ化（その際はトークンを OAuth 方式に変更推奨）

## 注意

- これは個人用ローカルツールの想定です。配布する場合はトークンの平文保存をやめ、OS のキーチェーンや OAuth デバイスフローへの変更を推奨します。
